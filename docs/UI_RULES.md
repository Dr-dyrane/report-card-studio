# UI Rules

Kradle should feel calm, premium, inevitable, and easy to scan. The product should read like a finished tool, not a collection of decorated screens.

## Core Direction

### Fewer Visible Controls
- Reduce simultaneous choice on each screen.
- Prefer strong defaults over multiple competing actions.
- Let context decide the likely next step whenever possible.

### Hierarchy Through Spacing, Not Decoration
- One hero per screen.
- One primary action per surface.
- Everything else should be quieter.
- Use spacing, grouping, and density before adding color or chrome.

### Calmer Data Density
- Tables should feel editorial, not dashboard-heavy.
- Reduce pill noise.
- Avoid surfaces inside surfaces unless the nesting improves clarity.
- Keep dense screens compact, readable, and rhythmically aligned.

### Continuity In Motion
- Sheets, modals, and transitions should feel like one system.
- Motion should be restrained, almost invisible, but consistent.
- Avoid playful animation. Favor calm fades and short vertical easing.

### Clearer Product Voice
- More confidence.
- Less labeling.
- Less explanatory text.
- UI copy should feel obvious and direct.
- Aim for "of course that's where it is."

### Ruthless Simplification
- Remove cards that do not earn their space.
- Collapse related settings into fewer grouped decisions.
- Turn some visible page actions into context defaults instead.
- If a surface can be simpler, it should be.

### Perfect Output Artifacts
- The report card PDF is a flagship object.
- Print and export should feel as designed as the app itself.
- Export artifacts should never feel like screenshots of the web UI.

### Detail Quality
- Icon optical balance matters.
- Baseline alignment matters.
- Row rhythm matters.
- Hover and press states should feel intentional.
- Empty states should be quiet.
- Typography should be consistent across pages and surfaces.

## Kradle-Specific Rules

### Semantic Mood
- Meaning should shape emphasis.
- Use tonal mood surfaces for focus, success, warning, and danger.
- Keep them borderless and blended into the material system.
- Mood should guide hierarchy, not become loud status decoration.

### Theme Discipline
- Light mode should stay clean enough for mood surfaces to read.
- Dark mode should feel vivid and deep, not washed out.
- Background atmosphere should support the UI, not compete with it.

### Surface Design
- Favor flatter, denser surfaces over bulky stacked cards.
- Reduce haze and overblown shadow spread.
- Preserve material depth without making the interface foggy.

### Mobile Pattern
- Prefer blade cards with focused sheets for secondary detail.
- Use progressive disclosure instead of long open-form pages.
- Keep utility actions and account actions separate from primary navigation.

### Document Pattern
- PDF and print layouts should be document-first.
- Remove sections that the current data model cannot truthfully render.
- Favor a compact, elegant academic sheet over decorative export cards.

## Review Checklist

Before calling a UI pass finished, check:
- Is there one clear hero?
- Is there one clear primary action?
- Can any visible control disappear without harming clarity?
- Are the tonal surfaces readable in both light and dark mode?
- Does the page feel calmer when zoomed out?
- Are tables and lists aligned, dense, and easy to scan?
- Do overlays feel like one family?
- Does the PDF feel like a flagship artifact?
- Is any copy still explaining itself instead of simply being clear?
